composer-dependency-triage

● available dependencies security upgrade v0.1.0 MIT

A Laravel-native dependency triage advisor. Point it at any Laravel project and it turns raw Composer output into a prioritized, ready-to-act maintenance plan — without ever editing your project.

What it does

Both composer outdated and composer audit show you facts: what's out of date, what's vulnerable, what's abandoned. They don't tell you what to do, or in what order. This skill is the decision layer on top of them.

  • ▹
    Reads real tool output as ground truth — never invents a version number or a CVE.
  • ▹
    Classifies every direct dependency: 🔴 vulnerable · 🟠 abandoned · 🟡 major · 🟢 safe · ✅ healthy.
  • ▹
    Recommends maintained replacements for abandoned packages — even when Composer just prints "No replacement was suggested" — with the migration shape (drop-in vs. API change).
  • ▹
    Finds the safest verified fix for a vulnerability: reads each advisory range literally, picks the lowest fixed version, prefers a drop-in within the current major, and confirms that version actually exists — so you're never pushed into a needless major migration.
  • ▹
    Checks PHP/Laravel compatibility before any major and routes framework-coupled bumps into the upgrade path.
  • ▹
    Scores breakage risk with dry-run + one-lane-at-a-time guidance, so an update can't silently take your app down.
Advise-only. It reads files and runs read-only commands (composer outdated, audit, why), then hands you a plan. It never edits your composer.json, lockfile, or code — you run every command yourself.

Example output

A real triage run, condensed. Hard facts come from the tools; the plan and replacement advice are the skill's judgment — each claim verified before it's printed.

phpseclib/phpseclib  2.0.30 → 2.0.54   7 advisories  drop-in within 2.x — NOT a 3.x migration

laravelcollective/html → spatie/laravel-html   (API change — audit your Blade forms)

## Action plan
✅ Do now        verified drop-in security patches + safe bumps
⚠️ Do carefully  swap abandoned packages; plan the Laravel upgrade
🛑 Defer         framework-coupled majors, sequenced into the upgrade

The three lanes

✅ Do now

Vulnerabilities fixed by a verified drop-in, plus safe minor/patch bumps. Run the commands, run your tests.

⚠️ Do carefully

Abandoned swaps and majors that need code edits. Read the upgrade guide first; do one at a time.

🛑 Defer

Risky majors blocked on a Laravel/PHP upgrade. Sequenced into a dedicated pass.

Install & use

Add the library to Claude Code

Run /plugin marketplace add ArtemProshkovskiy/laravel-maintenance-skills then /plugin install laravel-maintenance@laravel-maintenance-skills.

Open Claude Code in your project

Start it inside the folder that has composer.json.

Ask in plain language

"audit my composer dependencies", "are any of my packages abandoned?", "check my dependencies before upgrading Laravel" — the skill activates automatically.

Review, then run

You get a Markdown plan with exact commands. Nothing is changed for you — run each on a branch, with tests.

Boundaries

  • does not modify composer.json, composer.lock, or any code;
  • does not run composer require/update/remove — you run those;
  • does not invent versions or CVEs — only what the real tools returned;
  • does not tell you to update a transitive dependency directly — it explains the chain via composer why and points you at the direct parent;
  • does not recommend a major your PHP/Laravel can't satisfy — it marks it "blocked on upgrade".

It is not a security scanner: vulnerability facts come from composer audit (the PHP Security Advisories database), not a CVE database of its own. A clean audit means "nothing found", not "guaranteed secure".
